# TWO POINTER TECHNIQUE — 5 LEVEL ROADMAP (25 QUESTIONS)

---

# LEVEL 1 — FOUNDATIONS (Basic Traversal + Opposite Pointers)

**Goal:** Understand pointer movement and conditions

### Problems

1. Reverse an array in-place
2. Check if a string is a palindrome
3. Valid palindrome (ignore non-alphanumeric characters)
4. Remove duplicates from sorted array
5. Merge two sorted arrays (in-place or extra space)

### Focus

* Basic pointer initialization (`left`, `right`)
* Movement rules
* Boundary handling

---

# LEVEL 2 — PAIR MATCHING & CONDITIONS

**Goal:** Learn decision-based pointer movement

### Problems

6. Two Sum II (sorted array)
7. Find pair with given difference
8. Container With Most Water
9. 3Sum (basic version)
10. 3Sum Closest

### Focus

* Conditional pointer shifts
* Sorting + two pointers
* Avoiding duplicates

---

# LEVEL 3 — FAST & SLOW POINTERS (Cycle & Structure)

**Goal:** Master same-direction pointers

### Problems

11. Detect cycle in linked list
12. Find middle of linked list
13. Remove Nth node from end
14. Happy Number (cycle detection)
15. Find duplicate number (Floyd’s cycle)

### Focus

* Fast vs slow logic
* Cycle detection
* Pointer math reasoning

---

# LEVEL 4 — PARTITIONING & IN-PLACE TRANSFORMATIONS

**Goal:** Learn rearrangement and partition logic

### Problems

16. Sort Colors (Dutch National Flag)
17. Move zeroes to end
18. Partition array (even/odd or < pivot)
19. Remove duplicates II (allow limited duplicates)
20. Trapping Rain Water

### Focus

* In-place swaps
* Partition invariants
* Multi-pointer coordination

---

# LEVEL 5 — ADVANCED / HYBRID PATTERNS

**Goal:** Combine two pointers with other techniques

### Problems

21. 4Sum
22. Subarray with given sum (positive integers)
23. Minimum window substring (two pointers + sliding window)
24. Longest substring without repeating characters
25. Boats to Save People

### Focus

* Combining with hashing or sliding window
* Multi-condition movement
* Optimization strategies

---

# HOW TO APPROACH EACH PROBLEM

For every question, follow this exact thinking:

1. Identify pointer type:

   * Opposite (left–right)
   * Same direction (fast–slow)

2. Define invariant:

   * What condition must always remain true

3. Decide movement:

   * When to move left
   * When to move right

4. Optimize:

   * Avoid re-checking
   * Handle duplicates

# FINAL PROGRESSION OUTCOME

If you complete all 25:

* You will understand **all core two-pointer variations**
* You will handle:

  * Arrays
  * Strings
  * Linked lists
* You will be able to **derive solutions instead of memorizing**
